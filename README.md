# Legal CRM Automation Portfolio

Портфолио-проект, который показывает не “ещё одну CRM-табличку”, а законченный сценарий автоматизации юридической практики: лид попадает в систему, классифицируется по типу дела, получает дедлайн, чеклист задач, ответственного и Telegram-уведомления.

Это демонстрационный продукт-витрина: компактный, но собран как реальное приложение, которое можно запустить локально, задеплоить на Vercel и подключить к Supabase.

Технический onboarding для разработчика или AI-агента лежит здесь: [docs/AGENT_ONBOARDING.md](docs/AGENT_ONBOARDING.md).

## Что показывает этот проект

Я умею собирать не просто интерфейс, а рабочий end-to-end процесс:

- продумать бизнес-сценарий;
- сделать понятный UI;
- подключить serverless API;
- вынести секреты на сервер;
- добавить базу данных;
- настроить Telegram-уведомления;
- предусмотреть demo-mode;
- подготовить проект к деплою на Vercel;
- довести адаптивность до состояния, где приложением можно пользоваться на desktop, ноутбуке и мобильном экране.

Внутри проекта нет настоящих персональных данных. Все клиенты, телефоны и дела — демонстрационные.

## Основная идея

Юридические заявки часто теряются не потому, что нет CRM, а потому что нет процесса:

- кто отвечает за дело;
- какой дедлайн;
- какие документы собрать;
- что уже сделано;
- нужно ли уведомить юриста;
- когда клиент перешёл в другой статус.

Проект решает именно эту задачу. Клиент — это не строка в таблице, а маленький автоматизированный workflow.

## Что реализовано

### CRM для клиентов и дел

- добавление и удаление клиентов;
- статусы: `Новый`, `В работе`, `Закрыт`;
- поиск по клиенту, телефону и категории;
- фильтры и счётчики;
- отдельные заметки по делу;
- inline-редактирование статуса, категории и дедлайна прямо в строке;
- выгрузка контактов и задач в Excel-совместимую таблицу.

### Категории дел

Есть базовые категории:

- ДТП / страховая;
- трудовой спор;
- семейное дело;
- бизнес / договор.

Категории можно добавлять вручную. У каждой категории есть тип клиента (Физлицо или Юрлицо), ответственный, типовой документ, базовый срок и чеклист задач.

Цветовое разделение сделано осознанно и управляется выбором типа категории: физические лица — розовые категории, юрлица — фиолетовые. Цвета намеренно отличаются от палитры статусов (синий/жёлтый/зелёный), чтобы категория и статус клиента не путались визуально.

Клиенты с горящим дедлайном или срочными делами (по ключевым словам в заметке или дедлайну ближе 12 часов) помечаются бейджем «Срочно» прямо у имени.

### Задачи по клиенту

У каждого клиента есть список задач:

- задача имеет название;
- задачу можно отметить выполненной;
- у задачи есть дедлайн через календарь;
- дедлайн ближайшей открытой задачи показывается прямо в таблице и обновляется сам раз в минуту (статус "Просрочено" не зависает);
- завершение задачи может отправлять Telegram-уведомление;
- порядок задач можно менять: перетаскиванием (desktop) или кнопками ▲▼ (везде, включая мобильные — нативный drag-and-drop на тачскринах ненадёжен).

Это важно: задачи не захардкожены только для красоты. Их можно добавлять, менять, переупорядочивать и закрывать.

### Автоматизация

При создании клиента приложение подбирает категорию, ответственного, документ, дедлайн и список задач. В demo-сценариях это позволяет показать, как сервис помогает юристу не держать процесс в голове.

Кнопка `Запустить демо` добавляет новые демонстрационные дела с разными категориями, задачами и стадиями выполнения. Это сделано для портфолио: можно быстро показать живой поток заявок без настоящей базы клиентов.

### Экспорт в Excel

Контакты и задачи можно выгрузить в Excel-совместимую таблицу. Экспорт формируется так, чтобы каждая строка содержала клиента и конкретную задачу: имя, телефон, категорию, тип клиента, статус, заметку, задачу, статус задачи и дедлайн.

Это полезно для передачи списка дел руководителю, юристу или для быстрой аналитики без доступа к самой CRM.
### Telegram-уведомления

Поддержаны уведомления:

- о новом клиенте;
- о смене статуса клиента;
- о выполнении задачи.

Для уведомлений есть флаги на уровне клиента:

- уведомлять о смене статуса;
- уведомлять о выполнении задач.

Telegram подключён через Vercel serverless endpoint, поэтому токен бота не попадает во фронтенд.

### Supabase и demo-mode

Supabase используется как основное хранилище через serverless API `/api/clients`.

Если Supabase или API недоступны, приложение не ломается. Оно явно переходит в `Demo mode` и хранит тестовые данные в `localStorage`. Это сделано специально:

- локальный фронтенд можно открыть без настройки базы;
- портфолио можно показать даже без backend-секретов;
- на Vercel при наличии переменных окружения работает live-режим.

### Адаптивность

Интерфейс переработан под разные ширины:

- на desktop таблица использует всю ширину экрана;
- дублирующий правый блок убран, чтобы не съедать место;
- на ноутбуках и планшетах таблица не обрезает кнопки и дедлайны;
- на узких экранах строки превращаются в карточки без горизонтальной прокрутки;
- кнопка удаления клиента живёт в шапке карточки (рядом с именем), а не отдельной строкой внизу;
- формы, задачи, фильтры и категории перестраиваются в один столбец на мобильных.

Это не просто “медиа-запрос для галочки”: структура UI изменена так, чтобы CRM оставалась usable.

## Почему так сделано

### Почему Vercel serverless API

Проект должен легко деплоиться. Vercel позволяет держать фронтенд и API рядом, без отдельного backend-сервера. Для портфолио это хороший баланс: видно, что есть backend-логика, но проект остаётся компактным.

### Почему Supabase

Supabase даёт быструю PostgreSQL-базу и нормальную модель хранения. При этом ключи не используются на клиенте: фронт ходит только в `/api/clients`, а API уже работает с Supabase.

### Почему есть demo-mode

Портфолио должно открываться даже тогда, когда у проверяющего нет `.env.local`, Supabase-таблицы и Telegram-токена. Поэтому деградация сделана явной и управляемой.

### Почему Telegram

Telegram показывает реальную автоматизацию вокруг CRM. Это не декоративная кнопка, а сценарии уведомлений по событиям: новый лид, смена статуса, выполненная задача.

## Быстрый запуск

Только фронтенд:

```bash
npm install
npm run dev
```

В этом режиме Vite обычно не поднимает `/api/*`, поэтому приложение может перейти в demo-mode.

Запуск ближе к Vercel:

```bash
npm install
npm run dev:vercel
```

Для production-сборки:

```bash
npm run build
```

## Деплой

Проект рассчитан на Vercel:

- Build command: `npm run build`
- Output directory: `dist`
- Serverless API: папка `api/`
- Environment Variables: Supabase и Telegram-секреты

Подробности по env, SQL-схеме, API и подводным камням: [docs/AGENT_ONBOARDING.md](docs/AGENT_ONBOARDING.md).
