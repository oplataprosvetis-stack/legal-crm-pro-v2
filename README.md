# Legal CRM — прототип для юриста

Рабочий MVP: мини-CRM для ведения клиентов юриста.  
Собран за несколько часов как тестовое задание для «Доступного права».

## Что реализовано

- Таблица клиентов с кликом на строку для раскрытия деталей
- Добавление клиента: имя, телефон (с автоматической маской `+7(ХХХ)-ХХХ-ХХ-ХХ`), статус, заметка
- Изменение статуса прямо в таблице (Новый → В работе → Закрыт)
- Визуальный прогресс-индикатор этапов дела
- Счётчики по статусам + фильтрация по кнопке
- Поиск по имени и телефону
- Заметки по делу с авто-сохранением
- Toast-уведомления при действиях
- Данные сохраняются в localStorage
- Бонус: Telegram-уведомления о новых клиентах — каждый пользователь подключает свой аккаунт по `@username`, кнопка в шапке ведёт на бота

## Стек

React + Vite + localStorage. Без бэкенда, без БД — максимально быстрый MVP.  
Telegram-уведомления — через Vercel serverless functions (`api/connect.js`, `api/notify.js`).

## Запуск локально

Обычный `npm run dev` поднимает только фронтенд — папка `api/` (serverless-функции) при этом не исполняется, и Telegram-уведомления работать не будут (запросы на `/api/*` вернут 404).

**Если нужен только фронтенд (без Telegram):**
```bash
npm install
npm run dev
```

**Если нужно протестировать и Telegram-уведомления — через Vercel CLI:**
```bash
npm install -g vercel
vercel login             # один раз, откроет браузер для входа
vercel link               # привязать папку к проекту на Vercel
vercel env pull .env.local  # подтянуть переменные окружения (включая TELEGRAM_BOT_TOKEN)
vercel dev                 # поднимет и фронт, и /api — как на проде
```
Сервер поднимется на `http://localhost:3000/`.

## Деплой на Vercel

Настроено автоматически: любой `git push` в ветку `main` репозитория на GitHub триггерит пересборку и публикацию на Vercel (через GitHub-интеграцию проекта). Ручного деплоя не требуется.

Если разворачиваете с нуля:
1. Загрузить проект на GitHub
2. Зайти на vercel.com → New Project → выбрать репозиторий
3. Build command: `npm run build`, Output directory: `dist`
4. Deploy

## Telegram-уведомления (бонус)

В настройках проекта на Vercel (**Settings → Environment Variables**) добавить:

- `TELEGRAM_BOT_TOKEN` — токен бота от @BotFather. Нужен для **Production**, **Preview**, и отдельно для **Development** (если планируете тестировать локально через `vercel dev` — см. выше). Важно: если переменная помечена как *Sensitive*, добавить её в Development не получится — Vercel такое не разрешает для sensitive-переменных. В этом случае удобнее завести отдельную (не sensitive) запись с тем же именем, но только для Development.
- `TELEGRAM_CHAT_ID` — опционально, дефолтный чат для уведомлений, если пользователь ещё не подключил свой Telegram через интерфейс приложения.

Если переменные не заданы — приложение работает, но без уведомлений (кнопка подключения в интерфейсе покажет ошибку `bot_not_configured`).

### Как пользователю подключить уведомления (без переменных)
1. В приложении нажать кнопку **"Подключить уведомления"** (шапка, синяя пилюля) → откроется панель с прямой ссылкой на бота
2. Написать боту любое сообщение в Telegram
3. Ввести свой `@username` в поле и нажать «Подключить» — приложение найдёт чат через `getUpdates` и запомнит `chat_id`

## Как создать Telegram-бота

1. Написать @BotFather в Telegram → /newbot → получить токен
2. Написать боту любое сообщение
3. Открыть `https://api.telegram.org/bot<TOKEN>/getUpdates` — найти `chat.id`
